# noet: архитектура конструктора

Статус: рабочий документ. Версия 1.0 (2026-06-10).
Связанные: [ROADMAP.md](ROADMAP.md), [FEDERATION.md](FEDERATION.md), концепция СЦЗ v0.3.

## Что мы строим

Не платформу и не сервис. Конструктор: набор примитивов, протоколов и инструментов,
из которых люди сами собирают форумы, блоги, игры, доски, каталоги и сообщества,
не меняя ядро. Мерило успеха: поверх noet построено больше, чем внутри noet.

Проверка каждого решения: «это строительный блок или платформа?» Если функция
собирается из существующих примитивов, новый специальный механизм запрещён.

## Пять примитивов и их реализация

Всё в noet сводится к пяти примитивам. Новые сущности выводятся из них.

### 1. Identity (личность)

Криптографический ключ (Nostr/schnorr). Не аккаунт на сервере.

| Часть | Реализация | Статус |
|---|---|---|
| Ключ | расширение (chrome.storage) или localStorage, NIP-07 `window.nostr` | есть |
| Профиль | kind 0 (имя, аватар, о себе) | есть |
| Хэндл | реестр: уникальный `handle` на экземпляр, имя `handle.me` | есть |
| Поручительства | kind 8001/8002, p-тег = за кого | есть |
| Доступы | kind 8010/8011: scope, срок, передача дальше, цепочка ≤3 | есть |
| Репутация | НЕ хранится. Выводится детерминированной функцией из событий | есть |
| Восстановление | бэкап-файл ключа. Социального восстановления нет (честно: потерял ключ, потерял личность) | частично |

Ключевое свойство: личность работает в любом экземпляре noet и в любом
Nostr-приложении мира. Это не наш замок
