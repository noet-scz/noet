Для разработчиков

Всё, что нужно, чтобы построить своё на noet: страницы, игры, доски, дашборды. Сервер для этого не нужен.

ПоискЛюдиАккаунт

window.noet — данные твоего приложения

На каждой странице, открытой через noet, доступен window.noet. Данные это подписанные события: они принадлежат автору, хранятся в общей сети и переживают любой отдельный сервер.

// кто я
const { pubkey } = await noet.me();

// сохранить запись (например, рекорд игры)
await noet.publish({ kind: 30078, tags: [["d","my-game-score"], ["t","my-game"]], content: "1240" });

// прочитать записи (лидерборд)
const events = await noet.query({ kinds: [30078], "#t": ["my-game"] });

window.nostr — подпись (NIP-07)

Стандартный интерфейс подписи. Ключ живёт в расширении и не попадает на страницу.

const pk = await window.nostr.getPublicKey();
const signed = await window.nostr.signEvent({ kind: 1, tags: [], content: "привет" });

Аналитика в одну строку

Подключи снипет, и события посещений попадут в твой дашборд (кнопка «Создать дашборд» в аккаунте).

<script src="https://noet-scz.github.io/noet/dist/noet-analytics.js" data-app="ТВОЙ_ТЕГ" data-auto></script>
// или вручную: noetTrack("click", { button: "play" })

HTTP API реестра

ЧтоЗапрос
Все именаGET /api/names
Имя → CIDGET /api/resolve/<имя>
ПоискGET /api/search?q=
Участники + репутацияGET /api/people
Доступы и правилаGET /api/gov
Полная копия (форк)GET /api/export
Кто яGET /api/me + Authorization: Bearer <токен>
Опубликовать страницуPOST /api/publish { title, body } или { mode:"html", body }
Задеплоить проектPOST /api/publish-dir { sub, files:[{path, data(base64)}] }
ПоручитьсяPOST /api/vouch { event: kind 8001, p=за кого } (8002 = отозвать)
Дать доступPOST /api/delegate { event: kind 8010, p=кому, scope, until?, redelegate? }
Отозвать доступPOST /api/revoke { event: kind 8011, e=id доступа }
Создать инвайтPOST /api/invite { event: kind 8020 } (нужен доступ «инвайты»)
Обновить страницу зоныPOST /api/publish-zone { name, body } (нужен доступ к имени)

События управления (8001–8020) это обычные подписанные Nostr-события. Реестр проверяет подпись и право, всё хранится открыто и попадает в /api/export.
