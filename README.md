# noet

**Цифровой конструктор, а не платформа.** Набор примитивов, из которых люди сами собирают
форумы, блоги, доски, вики, маркетплейсы и сообщества — не меняя ядро. Личность это
криптографический ключ, контент лежит в IPFS, имена раздаёт собственный реестр noet.

Показатель успеха: **поверх noet построено больше систем, чем встроено внутрь.**

## Как это работает

- **Клиент = браузерное расширение** (`ext/`). Перехватывает зарегистрированные имена
  `*.nt`/`*.me`, тянет контент из IPFS и рендерит его в sandbox-странице (где работает JS).
  Инжектит `window.nostr` (подпись, NIP-07) и `window.noet` (данные через реле).
- **Личность** — пара ключей Nostr, ключ живёт в расширении. Ни почты, ни паролей.
- **Контент** — в IPFS, адресуется по содержимому (CID одинаков везде).
- **Имена** — реестр сопоставляет имя координате. Это единственное, чему нужен «судья
  очерёдности»; сейчас сервер-судья (назван явно), цель — OpenTimestamps без сервера.
- **Управление** — доступы и репутация это подписанные события (см. `gov.mjs`,
  страница `people.nt`). Репутация даёт ёмкость, не власть. Полная копия всего: `/api/export`.

## Состав

| Компонент | Что |
|---|---|
| `ext/` | расширение: перехват имён, sandbox-рендер, window.nostr/noet, менеджер ключа |
| `server.mjs` | реестр: имена + проверка событий + Nostr-реле + seed-пиннинг + зеркало имён |
| `auth.mjs` | личность: проверка Nostr-событий, членство, сессии |
| `gov.mjs` | управление: поручительства, доступы, репутация-ёмкость (Policy-примитив) |
| `ws.mjs` | минимальное Nostr-реле (NIP-01/09) |
| `web/` | страницы реестра: поиск, личность/редактор, реле, люди, для разработчиков |
| `sites/` | примеры (манифест, дашборд, кликер) |
| `dist/` | конфиг и зеркало имён для расширения (GitHub Pages) |

## Запуск

Реестр: `node server.mjs` (нужен локальный kubo: IPFS_GW/IPFS_API). Сборка расширения:
`node build-ext.mjs` → `dist/noet-chrome.zip`, `dist/noet-firefox.xpi`.

`server.mjs` исторически двухролевой (по `REGISTRY_URL`), но демон-клиент ретайрен — клиент
теперь расширение.

## Свой экземпляр

Код открыт. Скачай копию состояния (`/api/export`), подними свой реестр — у тебя свой noet.
Первый участник нового экземпляра автоматически его владелец. Подробнее о синхронизации
между экземплярами: [docs/03-federation.md](docs/03-federation.md).
