# noet

Цензуроустойчивое сообщество. Личность: криптографический ключ, контент: в IPFS, имена: в собственном реестре noet.

## Как это работает

Контент адресуется по содержимому и лежит в IPFS. Имя (например `noet.nt`) это просто указатель на CID. Каждый пользователь ставит у себя **локальный резолвер** (демон): он берёт список имён с реестра, тянет контент напрямую из IPFS у себя на машине и отдаёт браузеру. Сервер хранит только имена. Если сервер выключат, уже известный контент продолжает открываться.

## Установка (Linux)

```
curl -fsSL https://noet-scz.github.io/noet/dist/install.sh | bash
```

Ставит kubo (IPFS) и демона как фоновые сервисы. Дальше один раз указать браузеру PAC и открывать `noet.nt` как обычный сайт. Подробности и другие платформы: https://noet-scz.github.io/noet/dist/

## Роли

Один файл `server.mjs`, две роли по наличию `REGISTRY_URL`:

- **реестр** (без `REGISTRY_URL`): хранит имена, личность (Nostr-ключи), Nostr-реле, seed-пиннинг. Контент в браузер не отдаёт.
- **демон** (`REGISTRY_URL=...`): на машине пользователя. PAC шлёт сюда `*.nt` и `*.me`. Резолвит имя в CID, тянет из локального kubo, отдаёт. Запись (вход, публикация, реле) проксирует на реестр.
